HTML

  • HTML,Hypertext Markup Language, is used to create the structure of a webpage. It is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is a type of content on a webpage such as: text elements, image elements, header elements, and list elements.
  • HTML elements are composed of an opening and closing tag represented with angle brackets. These tags contain everything we want to include in the element. This can contain content and other nested elements.
  • The head element contains information or metadata, that helps the browser to render the page correctly. Elements included within the head are not visible when the user looks at a webpage.
  • The body element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • We can combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • The git command must precede every git statement.
  • The checkout command is to move the working branch to a new branch.
  • git status: checks what branch we are currently in.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • The add command adds modifications in the current working branch to the staging area.
  • The -A flag indicates that we want to add all changes.

JavaScript

  • Variables are containers that store values.
  • Comments are snippets of text that can be added along with code.
  • An operator is a mathematical symbol that produces a result based on to values or variables.
  • Conditionals are code structures used to test if an expression returns true or not.
  • Event handlers are code structures that listen for activity in the browser and run code in response.
