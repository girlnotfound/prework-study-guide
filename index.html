HTML

  • HTML, Hypertext Markup Language, is used to create the structure of a webpage. It is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is a type of content on a webpage such as: text elements, image elements, header elements, and list elements.
  • An element refers to the entire element from opening tag to closing tag, A tag refers only to what appears inside the angle brackets.
  • HTML elements are composed of an opening and closing tag represented with angle brackets.
    • These tags contain everything we want to include in the element. This can contain content and other nested elements.
  • The head element contains information or metadata, that helps the browser to render the page correctly.
    • Elements included within the head are not visible when the user looks at a webpage.
  • The body element contains the content that is shown to the user.
    • All the elements that we want the user to see and interact with must be placed inside this element.
  • <h1> <h2> <h3> <h4> <h5> and <h6> are visible elements that represent the level of heading a given text block represents. Use heading tags in order from <h1> to <h6>
  • The <p> element represents a paragraph or block of text.
  • <ul> <ol> and <li> are elements which represent unordered(bulleted) lists, ordered(numbered) lists, and list items.
  • The <img> element contains information about images that are displayed on the webpage.
  • The <br> element creates a line of empty space, or a line break between two blocks of content

CSS

  • CSS, Cascading Style Sheets, is a stylesheet language used to define the color, aesthetics, visual layout, and organization of a webpage. CSS is used to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • CSS works by associating rules with HTML elements. CSS rules contain a selector and a declaration
    • Selectors indicates which element the rule applies to.
    • Declarations indicate how the element referred to in the selector should be styled.
  • CSS declarations sit inside curly brackets and are made up of a property and a value.
    • Properties indicate aspects of the element you want to change.
    • Values specify the settings you want to use for the chosen properties. For example, specifying a color propery, the value is the color you want the text in the element to be.
  • Examples of CSS properties include:
    • margin, which indicates how much space we want around the outside of an element.
    • padding, which indicates how much space we want around the content inside an element.
  • We can combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • Git is a software that allows you to keep track of changes made to a project over time. Git works by recording the changes you make to a project, storing those changes, and allowing you to reference them as needed.
  • A Git project can be thought of as having three parts.
    1. A Working Directory: where you will be doing all the work: creating, editing, deleting and organizing files.
    2. A Staging Area: where you will list changes you make to the working directory.
    3. A Repository: where Git permanently stores those changes as different versions of the project.
  • Git commands to use to help keep track of changes made to a project: (note: the git command must precede every git statement)
    • git status: checks what branch we are currently in.
    • git add: adds modifications in the current working branch to the staging area.
    • git add -A: to stage all changes in the repository.
    • git add commit -m: to create a new commit with the changes that have been added to the staging area. The -m is followed by a message in quotes describing what changes the commit contains.
    • git pull origin branch-name: is used to fetch changes from a remote repository's branch and merge them into your current branch.
    • git push origin branch-name: is used to upload the local repository content to a remote repository.
    • git checkout: to move from the working branch to a new branch.
    • git checkout -b branch-name: creates a new branch and switches to it.

JavaScript

  • JavaScript is a programming language that adds interactivity to your website.
  • Variables are containers that store values.
    • You start by declaring a variable with the "let" keyword.
  • Comments are snippets of text that can be added along with code.
  • An operator is a mathematical symbol that produces a result based on to values or variables.
  • Conditionals are code structures used to test if an expression returns true or not.
  • Event handlers are code structures that listen for activity in the browser and run code in response.
  • Control flow is the order in which a computer executes code in a script.
